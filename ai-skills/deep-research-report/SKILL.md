# Deep Research Report

Use this skill when a user wants more than a quick fact. The goal is a researched answer with traceable sources and a concise final synthesis.

## Workflow

1. Clarify the research question only if the scope is genuinely ambiguous.
2. Write a compact plan:
   - define the topic,
   - list independent research angles,
   - identify expected evidence types,
   - set a stop condition.
3. Choose research passes:
   - Use one comprehensive pass for ordinary research.
   - Use one pass per explicit comparison item.
   - Use one pass per independent geography, product, company, or technical subtopic only when separation is useful.
4. Research broadly first, then narrowly.
5. After each source batch, assess:
   - what was learned,
   - what is still missing,
   - whether more searching will likely improve the answer.
6. Stop when the answer is well supported. Do not search forever for perfection.
7. Synthesize into a final answer with:
   - direct answer or executive summary,
   - evidence-backed sections,
   - caveats and uncertainty,
   - source list.

## Search Budget

- Simple factual research: 2-3 searches.
- Complex research: up to 5 focused searches per research pass.
- Stop early if the last two searches repeat the same information.
- Prefer fewer high-quality sources over many weak ones.

## Citation Rules

- Give each unique URL one citation number.
- Reuse the same number when citing the same URL again.
- Keep citations close to the claim they support.
- End with a `Sources` section.
- Distinguish hard facts from estimates or interpretation.

## Report Shapes

For comparisons:

1. Summary judgment.
2. Short overview of each option.
3. Comparison table or criteria-based sections.
4. Recommendation and caveats.

For topic overviews:

1. What it is.
2. Key concepts.
3. Current state.
4. Tradeoffs, risks, or open questions.
5. Practical recommendation.

For lists or rankings:

1. State criteria.
2. List items with concise evidence.
3. Explain edge cases and exclusions.
